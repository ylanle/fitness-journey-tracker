What I'm building: For mini project 2 I plan to build a gym progress tracker with food intake, exercise and personal measurements tracking. 
Who it's for / why: This is a tool for gym goers who want to keep the majority of their gym progress/journey in the app. There are apps like "myfitnesspal" that tracks food, then "hevy" which helps you log your workout/reps but I have yet to find an app that combines all these features in a simple and effective way. It will track past lifts and solves the issue of knowing whether you're making progress with your fitness journey or not (numbers wise)
The state it tracks: what data does the tool need to remember? The website will remember daily nutrion entries (calories, protein, carbs, daily macros), workout entries (exercise nam, weight, sets, rep), and body measurement records (body weight, body fat percentage, measurement date). On top of those individual numbers, it will also track calculated statistics like total calories consumed, personal records, workout volume and weight changes over time.  
Core features: Log daily nutrion (calories, protein etc. for users to hit their daily nutritional goals), log workout reps (to help with progressive overload), track personal measurement data (weight, body fat%)
What I don't know yet: which JavaScript concepts will you need to figure out? I need to learn how to save the workout, nutrition and body measurement information in Javascript, automatically update the page when users add new entries, calculate things like personal record and workout progress, organize them and display the data cleanly and aesthetically, and save user data so when the page closes and refreshes it is still there?
